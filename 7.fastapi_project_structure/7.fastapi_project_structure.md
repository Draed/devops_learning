# 7.fastapi_project_structure

*Created on:* 2026-01-06

## Description

Re-organize code according to fastapi best practices, creating folder and files:
- routers : folder to store all api routes
- core : folder to store core api element (not depending on features) like logging and config loading (from .env)
- models : folder to store objects models 
- schemas : folder to store objects schemas (using pydantic)
- services : folder to store api service (api function and logics)
- dependencies : file for element of code that need to be shared accross api (logging instance, database connection instance)


## Prerequisites

- Have python 3 installed

## Other change from previous stage

- reorganize code (moving files to according folder)
- adding some specific folders (core, schemas)
- adding dependencies file in order to share element accross whole api

## Usage

- Set env variable (from example.env for quick test):
```
cp example.env .enc
source .env
```

- Deploy application using virtualenv :
```
python3 -m venv venv
source venv/bin/activate
python -m pip install -r requirements.txt
uvicorn app.main:app --host $HOST --port $PORT --reload
```

- Run test (in another terminal) :
```
python3 -m venv venv
source venv/bin/activate
python -m pytest tests/
```
